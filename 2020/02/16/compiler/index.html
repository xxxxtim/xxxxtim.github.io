Compiler VS Interpreter

Compiler Language

首先必須了解一下我們現在使用的程式語言 (Highlevel-language)要怎麼轉換成電腦看得懂的語言(Machine-code)

[補充]:高低階語言的區分

以下以C語言(編譯型語言)為例:

[補充]: 編譯型語言表示轉成電腦語言途中會用到編譯器(Compiler)

流程如下:

  1. source code
  2. preprocessor(預處理器):將靜態資源引入，包含macro的使用
  3. compiler
  4. object code(二進制-MachineCode)
  5. Linker:連結外面Library，例如系統所提供的API
  6. executables(執行檔)

以下面圖片為例，Compiler會根據不同的架構去編譯成不同的執行檔去執行

這裡可以把開發步驟想成是一位教台語的老師，A、B、C framwork分別是三位學習台語的日本學生、美國學生、非洲學生，老師必須根據不同學生的國籍去翻譯教學內容

  • 優點:
    編譯器會把所有編譯型別語先轉換成電腦語言再轉換成執行檔，程式碼執行速度上會比直譯型別語言來的快

  • 缺點:
    在編譯流程中，必須和你所使用的CPU架構對應，這樣才能在電腦上運行，所以編譯型語言的缺點就是不適合跨平台

Interpreter Language

Interpreter 顧名思義就是直譯器，在這邊可以想成翻譯蒟蒻

以下以JavaScript語言(直譯型語言)為例

流程如下:

  1. Source code
  2. Interpreter

原理:
執行程式時都要先轉成另一種語言再作執行，因此直譯器的程式運行速度比較緩慢。它不會一次把整個程式轉譯出來，而是每轉譯一行程式敘述就立刻執行，然後再轉譯下一行，再執行，如此不停地進行下去。

這裡可以把開發步驟想成是一位教台語的老師，A、B、C framwork分別是三位學習台語的日本學生、美國學生、非洲學生，老師必須根據不同學生的國籍去翻譯教學內容，每位學生有一台翻譯機，可以即時翻譯老師的上課內容

  • 優點:
    可以跨平台執行並且消除編譯整個的負擔

  • 缺點:
    程式執行效率不好
