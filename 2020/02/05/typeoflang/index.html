想知道何謂弱型別與強型別語言嘛？？

電腦如何讀取指令

在解釋型別之前我們要先理解電腦是如何讀取指令
首先我們先看下面這張圖


CPU讀取指令流程如下：

  1. 資料由inputdevice(ex:鍵盤)輸入
  2. 經由databus傳輸，並下達指令（instruction）給CPU
  3. CPU會把得到的指令統一放在InstructionRegister
  4. ControlUnit會把指令讀取出來並進行解碼（Decode）
  5. 解碼成machinecode(電腦看的懂的語言ex:010001)
  6. 這裡CPU才真正知道到底要做哪些事情
  7. 執行指令
  8. 根據不同的指令到Memory去進行記憶體存取並進行運算（ALU執行）
  9. 將運算結果寫回Register(暫存器)

在此先把電腦讀取指令會使用到記憶體這觀念先記起來

型別的用意主要是和電腦講要借多大的記憶體去做存取

弱型別語言VS強型別語言

介紹之前先說明4個名詞

  • Static typed languages(靜態語言)
  • Dynamically typed languages（動態語言）
  • Strongly typed languages（強型別語言）
  • Weakly typed languages（弱型別語言）

Static typed languages(靜態語言)：

如果在編寫程式時（而不是在run time）知道變量的類型，則該語言為靜態類型。

ex:Java，C，C ++，FORTRAN，Pascal和Scala。

在靜態類型語言中，一旦使用類型聲明了變量，就無法將其分配給其他不同類型的變量，編譯時會產生錯誤

變數一開始宣告的型別必須與執行後的變數型別一致

以下為C語言的範例

Dynamically typed languages（動態語言）：

如果在運行時(run time)檢查了變量的類型，則語言是動態類型的。
ex:JavaScript，PHP，Python。

在動態類型語言中，變量在運行時通過賦與型別綁定到對象，並且可以在程序執行期間將相同的變量綁定到不同類型的對象。

以下為javascript的範例

Strongly typed languages（強型別語言）：

一個強類型語言是一個特定變量綁定到特定的數據，會導致類型錯誤。

以下為python的範例

相同型別只能和相同型別做處理，如果型別不一致只能利用轉型parse來處理

Weakly typed languages（弱型別語言）：

一個弱類型語言是一個特定變量綁定到特定的數據，不會導致類型錯誤。

結論：

  1. 型別的用意主要是和電腦講要借多大的記憶體去做存取
  2. 靜態語言：一開始要先宣告型別
  3. 動態語言：一開始不用先宣告型別
  4. 強型別語言：型別要一致才能做處理
  5. 弱型別語言：型別不用一致能做處理
  6. 強弱型別區分和靜態動態區分沒有任何關係
    ex: 強型別語言不見得是靜態語言，ex:C語言主要是靜態語言，可是他是弱型別
